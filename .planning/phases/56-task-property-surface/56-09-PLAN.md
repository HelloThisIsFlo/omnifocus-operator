**Canonical capture pattern** (`uat/capture_golden_master.py`):
- Scenarios are declared as dicts in `_build_scenarios()` with keys: `folder`, `file`, `scenario`, `description`, `operation` (`add_task` or `edit_task`), `params` (static dict) OR `params_fn` (lambda, for dynamic IDs), `capture_id_as` (optional — stores the returned task ID under a symbolic key for later reuse), `followup` (optional — a second operation chained on the captured ID, same shape).
- Example of a simple add scenario (from `01-add/01_inbox_task`):
  ```python
  {
      "folder": "01-add",
      "file": "01_inbox_task",
      "scenario": "01-add/01_inbox_task",
      "description": "Create a basic task in inbox with no optional fields",
      "operation": "add_task",
      "params": {"name": "GM-InboxTask"},
      "capture_id_as": "inbox_task",
  },
  ```
- Example of a chained add+edit scenario (from `02-edit/01_rename`):
  ```python
  {
      "folder": "02-edit",
      "file": "01_rename",
      "scenario": "02-edit/01_rename",
      "description": "Rename a task",
      "operation": "add_task",
      "params": {"name": "GM-RenameTarget"},
      "capture_id_as": "rename_target",
      "followup": {
          "operation": "edit_task",
          "params_fn": lambda: {
              "id": TASK_IDS["rename_target"],
              "name": "GM-RenameTarget-Renamed",
          },
      },
  },
  ```
