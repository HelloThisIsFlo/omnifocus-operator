4. Add `@field_validator("include")` with educational error message for invalid group names (FSEL-04). Follow the existing `@field_validator` pattern used for frequency type/interval validators — clean, agent-friendly messages. The Pydantic Literal type already rejects invalid values, but the raw Literal rejection message is not agent-friendly. Add a `@field_validator` that catches the validation and rewrites to an educational message listing valid groups:
```python
@field_validator("include", mode="before")
@classmethod
def _validate_include(cls, v: list[str] | None) -> list[str] | None:
    if v is None:
        return v
    valid = {"notes", "metadata", "hierarchy", "time", "*"}
    invalid = [g for g in v if g not in valid]
    if invalid:
        raise ValueError(
            f"Unknown field group(s): {', '.join(repr(g) for g in invalid)}. "
            f"Valid groups: notes, metadata, hierarchy, time, *"
        )
    return v
```

Note: The `@field_validator` with `mode="before"` runs BEFORE Pydantic's Literal validation, allowing us to produce a clean message. Check whether this pattern works with the existing `QueryModel` base class. If `mode="before"` doesn't intercept Literal validation, use `mode="wrap"` or accept the raw list[str] type and validate manually.

def shape_list_response_strip_only(result: ListResult, ...) -> dict[str, Any]:
