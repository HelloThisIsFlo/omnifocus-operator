<objective>
Fix ghost task bug: tasks with `effectiveDateCompleted` set (inherited from completed parent) but no own `dateCompleted` incorrectly report `availability: "available"`. Fix both the Python mapper (get_all path) and SQL filter clauses (list path) to use effective date columns. Same fix for projects.

Purpose: Ghost tasks pollute default queries (354 tasks returned instead of current active instance). Pre-aligns with v1.3.2 direction where all date filters use effective values.
Output: Fixed availability mapping in hybrid.py and query_builder.py, new test coverage for effective-only scenarios.
</objective>
