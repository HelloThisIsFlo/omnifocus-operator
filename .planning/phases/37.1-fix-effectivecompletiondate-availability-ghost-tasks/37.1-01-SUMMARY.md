# Phase 37.1 Plan 01: Fix effectiveDateCompleted Availability Ghost Tasks Summary

Fixed ghost task bug where tasks/projects with inherited completion or drop dates (effective-only, no own date) incorrectly reported availability "available" instead of "completed"/"dropped" in both the get_all Python mapper path and SQL list query path.

## One-liner

Availability mappers and SQL clauses now use effectiveDateCompleted/effectiveDateHidden, eliminating ghost tasks from default queries.
