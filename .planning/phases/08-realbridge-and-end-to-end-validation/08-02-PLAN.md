  <action>
    **1. Add SAFE-01 CI grep step to `.github/workflows/ci.yml`:**

    Insert a new step **before** the "Test" step:

    ```yaml
    - name: Safety check (SAFE-01)
      run: |
        if grep -r "RealBridge" tests/ --include="*.py" -l; then
          echo "ERROR: Test files must not reference RealBridge (SAFE-01)"
          echo "Use InMemoryBridge or SimulatorBridge for automated testing."
          exit 1
        fi
    ```

    This step greps all test files for the string "RealBridge". If any match is found, the build fails with a clear message. This is the CI half of the belt-and-suspenders safety approach (the runtime half is the factory guard from Plan 01).

    **2. Configure pytest `testpaths` in `pyproject.toml`:**

    Add `testpaths` to `[tool.pytest.ini_options]` to restrict pytest discovery to the `tests/` directory, explicitly excluding `uat/`:

    ```toml
    testpaths = ["tests"]
    ```

    This ensures `uv run pytest` never collects scripts from `uat/`.

    **3. Configure coverage omit in `pyproject.toml`:**

    Add `omit` to `[tool.coverage.run]` to exclude the simulator `__main__.py` (tested via subprocess, not importable unit tests) and the `__main__.py` entry point:

    ```toml
    [tool.coverage.run]
    source = ["omnifocus_operator"]
    omit = [
        "*/simulator/__main__.py",
        "*/__main__.py",
    ]
    ```

    This prevents the coverage threshold from failing due to modules that are tested via subprocess integration tests rather than direct imports.

    **4. Update CLAUDE.md with SAFE-01/02 rules:**

    Append a new section to `CLAUDE.md`:

    ```markdown
    ## Safety Rules

    - **SAFE-01**: No automated test, CI pipeline, or agent execution may touch `RealBridge`. All automated testing MUST use `InMemoryBridge` or `SimulatorBridge` exclusively. The bridge factory (`create_bridge("real")`) raises `RuntimeError` when `PYTEST_CURRENT_TEST` is set. CI enforces this via grep.
    - **SAFE-02**: `RealBridge` interaction is manual UAT only, performed by the human user against their live OmniFocus database. UAT scripts live in `uat/` and must NEVER be run by agents or CI. The `uat/` directory is excluded from pytest discovery and CI execution.
    ```

    **5. Verify TEST-02 (full pipeline via InMemoryBridge):**

    The existing test `tests/test_server.py::TestARCH01ThreeLayerArchitecture::test_list_all_returns_data_through_all_layers` already tests the full MCP -> Service -> Repository -> InMemoryBridge pipeline. Run it to confirm it passes:

    ```bash
    uv run pytest tests/test_server.py::TestARCH01ThreeLayerArchitecture -x --no-cov
    ```

    This satisfies TEST-02. No new test needed -- the existing test exercises the complete pipeline with zero OmniFocus dependency.
