    **Parser -- parse_rrule(rule_string: str) -> dict[str, Any]:**
    - "FREQ=DAILY" -> {"type": "daily"}
    - "FREQ=DAILY;INTERVAL=3" -> {"type": "daily", "interval": 3}
    - "FREQ=WEEKLY;BYDAY=MO,WE,FR" -> {"type": "weekly", "on_days": ["MO", "WE", "FR"]}
    - "FREQ=WEEKLY" -> {"type": "weekly"}
    - "FREQ=MONTHLY" -> {"type": "monthly"}
    - "FREQ=MONTHLY;BYDAY=2TU" -> {"type": "monthly_day_of_week", "on": {"second": "tuesday"}, "interval": 1}
    - "FREQ=MONTHLY;BYDAY=-1FR" -> {"type": "monthly_day_of_week", "on": {"last": "friday"}, "interval": 1}
    - "FREQ=MONTHLY;BYMONTHDAY=15" -> {"type": "monthly_day_in_month", "on_dates": [15], "interval": 1}
    - "FREQ=MONTHLY;BYMONTHDAY=-1" -> {"type": "monthly_day_in_month", "on_dates": [-1], "interval": 1}
    - "FREQ=YEARLY" -> {"type": "yearly"}
    - "FREQ=MINUTELY;INTERVAL=30" -> {"type": "minutely", "interval": 30}
    - "FREQ=HOURLY;INTERVAL=2" -> {"type": "hourly", "interval": 2}

    **Parser -- parse_end_condition(rule_string: str) -> dict[str, Any] | None:**
    - "FREQ=WEEKLY;COUNT=10" -> {"occurrences": 10}
    - "FREQ=MONTHLY;UNTIL=20261231T000000Z" -> {"date": "2026-12-31T00:00:00Z"}
    - "FREQ=DAILY" -> None

    **Create `src/omnifocus_operator/rrule/parser.py`:**
    Port and extend the spike parser (`.research/deep-dives/rrule-validator/rrule_validator.py`). Key differences from spike:

    1. Returns `dict[str, Any]` matching FrequencySpec model shapes (not RRuleComponents dataclass)
    2. Supports MINUTELY and HOURLY (per D-04) -- simple `{"type": "minutely/hourly", "interval": N}`
    3. BYDAY positional prefix parsing (per D-05): regex `r'^(-?\d+)?(MO|TU|WE|TH|FR|SA|SU)$'`
       - With prefix + FREQ=MONTHLY -> `monthly_day_of_week` with `on: {ordinal: day_name}`
       - Without prefix + FREQ=WEEKLY -> `weekly` with `on_days: [day_codes]`
       - Without prefix + FREQ=MONTHLY -> ValueError (per D-05)
    4. BYSETPOS rejection with educational error (per D-05): `"BYSETPOS is not supported. OmniFocus uses positional BYDAY format (e.g., BYDAY=2TU for 'second Tuesday'). Please report this rule string: {rule_string}"`
    5. Malformed strings fail-fast with ValueError (per D-07)
    6. UNTIL format conversion: `YYYYMMDDTHHMMSSZ` -> ISO-8601 `YYYY-MM-DDTHH:MM:SSZ`
    7. Interval defaults to 1 -- only include in result dict when != 1 (to match model default)

    Mapping tables:
