    2. In `config.py`, create a `Settings(BaseSettings)` class:
       - Import `BaseSettings` from `pydantic_settings`
       - The class should model all 7 OPERATOR_* env vars
       - Use `model_config = SettingsConfigDict(env_prefix="OPERATOR_")` so pydantic-settings automatically reads OPERATOR_* env vars
       - Fields:
         - `log_level: str = "INFO"` (maps to OPERATOR_LOG_LEVEL)
         - `week_start: str = "monday"` (maps to OPERATOR_WEEK_START)
         - `repository: str = "hybrid"` (maps to OPERATOR_REPOSITORY)
         - `ipc_dir: str | None = None` (maps to OPERATOR_IPC_DIR)
         - `bridge_timeout: float = 10.0` (maps to OPERATOR_BRIDGE_TIMEOUT)
         - `sqlite_path: str | None = None` (maps to OPERATOR_SQLITE_PATH; None means use default)
         - `ofocus_path: str | None = None` (maps to OPERATOR_OFOCUS_PATH; None means use default)

    3. Add a module-level `_settings` cached instance pattern:
       ```python
       _settings: Settings | None = None

       def get_settings() -> Settings:
           global _settings
           if _settings is None:
               _settings = Settings()
           return _settings
       ```
       This ensures Settings is instantiated once (reads env vars once at first access). Add a `reset_settings()` function for testing that sets `_settings = None`.

    4. Refactor `get_week_start()` to delegate to Settings internally:
       ```python
       def get_week_start() -> int:
           raw = get_settings().week_start.lower()
           if raw not in WEEK_START_MAP:
               raise ValueError(f"Invalid OPERATOR_WEEK_START '{raw}' -- use 'monday' or 'sunday'")
           return WEEK_START_MAP[raw]
       ```
       This preserves the existing public API so `test_date_filter_contracts.py::TestGetWeekStart` still passes without changes.

    5. Keep all existing constants (DEFAULT_LIST_LIMIT, FUZZY_MATCH_*, SystemLocation, etc.) unchanged.
