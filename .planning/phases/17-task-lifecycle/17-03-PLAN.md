  <files>src/omnifocus_operator/service.py, tests/test_service.py</files>
  <behavior>
    - Test: No-op lifecycle (complete already-completed task, lifecycle only) should NOT contain "No changes specified" in warnings -- only the action-specific "already complete" warning
    - Test: Same-container move (ending to same parent, move only) should NOT contain "No changes detected" -- only the action-specific same-container warning
    - Test: Tag no-op (replace with identical tags, tags only) should NOT contain "No changes specified" -- only the action-specific replace no-op warning
    - Test: Genuinely empty edit (no lifecycle, no tags, no move, no fields) STILL produces "No changes specified" warning
    - Test: Genuine field no-op (name set to same value, no actions) STILL produces "No changes detected" warning
  </behavior>
  <action>
    Two fixes in `service.py`:

    1. **Line 285** (Guard 1 -- empty edit): Change the condition from:
       `if len(payload) == 1:`
       To:
       `if len(payload) == 1 and not warnings:`
       This suppresses the "No changes specified" generic warning when action-specific warnings (from lifecycle no-op or tag no-op) already exist in `warnings`.

    2. **Line 351** (Guard 2 -- field no-op): Change the condition from:
       `if is_noop and len(payload) >= 1:`
       To:
       `if is_noop and len(payload) >= 1 and not warnings:`
       This suppresses the "No changes detected" generic warning when action-specific warnings (from same-container move) already exist in `warnings`.

       IMPORTANT: When this guard is suppressed (warnings already exist from same-container move), the function should still return early -- the same-container move already appended its warning and is_noop is True, so there's nothing to send to the bridge. Add an early return for same-container move case:
       After suppressing the generic warning, if `is_noop` is True and `warnings` is non-empty, return TaskEditResult with success=True and warnings. This prevents falling through to the bridge call.

       More precisely, replace lines 351-364 with:
       ```python
       if is_noop:
           if not warnings:
               # Genuine field-level no-op -- add generic warning
               warnings.append(
                   "No changes detected -- the task already has these values. "
                   "If you don't want to change a field, omit it from the request."
               )
           # Either way, return early (action-specific warnings already present, or generic just added)
           logger.debug(
               "OperatorService.edit_task: no-op detected, all values match current state"
           )
           return TaskEditResult(
               success=True,
               id=spec.id,
               name=task.name,
               warnings=warnings,
           )
       ```

    In `tests/test_service.py`:

    1. Update `test_lifecycle_noop_suppresses_status_warning` (line 1643) to ALSO assert that "No changes specified" is NOT in warnings.

    2. Add test `test_noop_lifecycle_no_spurious_empty_edit_warning`: Send lifecycle="complete" on already-completed task (no field edits). Assert: (a) success True, (b) has "already complete" warning, (c) does NOT have "No changes specified", (d) warning count is exactly 1.

    3. Add test `test_noop_same_container_move_no_spurious_noop_warning`: Send move ending=same-parent (no field edits). Assert: (a) success True, (b) has same-container warning, (c) does NOT have "No changes detected", (d) warning count is exactly 1.

    4. Add test `test_noop_tags_no_spurious_empty_edit_warning`: Send replace tags with identical set (no field edits). Assert: (a) success True, (b) has replace no-op warning, (c) does NOT have "No changes specified", (d) warning count is exactly 1.

    5. Review `test_stacked_warnings_completed_noop` (line 940) and `test_stacked_warnings_dropped_noop` (line 946): These test editing a completed/dropped task with a name set to the same value. The status warning fires (lifecycle_handled is False), AND "No changes detected" fires. With our fix, "No changes detected" is suppressed when warnings exist. Update these tests: remove the assertion for "No changes detected", update warning count to 1 (only the status warning).

    6. Verify `test_edit_empty_payload` (line 1100) still passes -- genuinely empty edit with no actions and no fields should still produce "No changes specified" (warnings is empty at that point).

    7. Verify `test_noop_detection_same_name` (line 1105) still passes -- field-only no-op with no actions should still produce "No changes detected" (warnings is empty at that point).
