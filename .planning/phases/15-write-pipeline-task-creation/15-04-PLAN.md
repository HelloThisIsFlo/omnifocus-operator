2. **Timestamp parsing fix:**
   - Create a NEW function `_parse_local_datetime(value: str | None) -> str | None` that:
     - Returns None if value is None
     - Takes a timezone-naive ISO 8601 string (e.g., "2026-04-01T10:00:00.000")
     - Interprets it as local time using `datetime.fromisoformat(value).replace(tzinfo=ZoneInfo(localzone))` where localzone is determined once at module level via `tzlocal.get_localzone_name()` or `time.tzname` -- BUT per the debug findings, use the timezone offset AT THE STORED DATE (not current time) to handle DST. The correct approach: parse the naive datetime, then use `ZoneInfo` to localize it (which handles DST automatically): `datetime.fromisoformat(value).replace(tzinfo=ZoneInfo(key))` is WRONG because replace doesn't fold. Instead use the `datetime` module's timezone-aware construction. The simplest correct approach:
     ```python
     from zoneinfo import ZoneInfo
     import time

     _LOCAL_TZ = ZoneInfo(time.tzname[0]) if time.tzname[0] != time.tzname[1] else ZoneInfo("UTC")
     ```
     Actually, the most reliable approach for getting the system timezone as a ZoneInfo:
     ```python
     from datetime import datetime, timezone
     from zoneinfo import ZoneInfo

     # Get system IANA timezone name
     _LOCAL_TZ_NAME = time.tzname  # Not reliable for IANA name
     ```

     **Simplest correct approach** (no new dependencies): Use `datetime.astimezone()` which knows the system timezone:
     ```python
     def _parse_local_datetime(value: str | None) -> str | None:
         if value is None:
             return None
         naive = datetime.fromisoformat(value)
         # Attach system local timezone (handles DST based on the date itself)
         local_dt = naive.astimezone()  # Python assumes naive datetimes are local time for astimezone()
         # WRONG: astimezone() on a naive datetime assumes it's UTC in Python < 3.12
         ```

     **Actually, the correct approach for Python 3.12+** (which this project uses): A naive datetime passed to `.astimezone()` is treated as local time starting Python 3.6+. But there's a subtlety: `naive.astimezone()` first replaces tzinfo with the system timezone **using the current UTC offset**, not the offset at the naive datetime's date. This is a known Python issue.

     **The truly correct approach:**
     ```python
     from zoneinfo import ZoneInfo
     import time

     def _get_local_tz() -> ZoneInfo:
         """Get system timezone as ZoneInfo (handles IANA name lookup)."""
         # time.tzset() ensures TZ env var is respected
         # For macOS, we can read /etc/localtime symlink target
         import subprocess
         result = subprocess.run(["readlink", "/etc/localtime"], capture_output=True, text=True)
         # /etc/localtime -> /var/db/timezone/zoneinfo/Europe/London
         tz_name = result.stdout.strip().split("zoneinfo/")[-1]
         return ZoneInfo(tz_name)
     ```

     This is getting too complex. **Use the `tzlocal` library** -- it's a tiny dependency (already handles all edge cases), OR use a simpler approach:

     **Recommended approach (no new deps, correct for DST):**
     ```python
     def _parse_local_datetime(value: str | None) -> str | None:
         """Parse a timezone-naive ISO 8601 string as local time, convert to UTC."""
         if value is None:
             return None
         naive = datetime.fromisoformat(value)
         # Use fold-aware localization: assume the naive time IS the local wall clock time
         # datetime.astimezone() on naive datetimes: Python treats them as local time
         # but uses the CURRENT utc offset. We need the offset AT THE GIVEN DATE.
         #
         # Workaround: attach UTC, convert to local to find the offset, then reverse.
         # Actually simpler: use the system timezone from /etc/localtime on macOS.
         import pathlib
         tz_path = pathlib.Path("/etc/localtime").resolve()
         tz_name = str(tz_path).split("zoneinfo/")[-1]
         local_tz = ZoneInfo(tz_name)
         local_dt = naive.replace(tzinfo=local_tz)  # This correctly uses DST at that date
         utc_dt = local_dt.astimezone(timezone.utc)
         return utc_dt.isoformat()
     ```

     **Key insight:** `ZoneInfo` + `replace(tzinfo=...)` IS correct for this case because ZoneInfo handles DST-aware UTC offset lookup based on the datetime's date. `replace()` doesn't "fold" -- it attaches the timezone and the ZoneInfo object computes the correct UTC offset for that specific date.

     So the final implementation:
     ```python
     import pathlib
     from zoneinfo import ZoneInfo
     from datetime import timezone

     def _get_local_tz() -> ZoneInfo:
         tz_path = pathlib.Path("/etc/localtime").resolve()
         tz_name = str(tz_path).split("zoneinfo/")[-1]
         return ZoneInfo(tz_name)

     _LOCAL_TZ = _get_local_tz()

     def _parse_local_datetime(value: str | None) -> str | None:
         """Parse timezone-naive ISO string as local time, return UTC ISO 8601."""
         if value is None:
             return None
         naive = datetime.fromisoformat(value)
         local_dt = naive.replace(tzinfo=_LOCAL_TZ)
         utc_dt = local_dt.astimezone(timezone.utc)
         return utc_dt.isoformat()
     ```
