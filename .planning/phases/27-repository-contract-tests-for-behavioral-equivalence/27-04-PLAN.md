    The `initial_state.json` will now contain raw format data (with "status", "parent" as string). InMemoryBridge accepts this and stores it internally. When `_handle_get_all` converts internal state to raw format, it needs to handle the case where the internal data might already be partly raw (e.g., tags with "status" instead of "availability").

    **IMPORTANT:** The initial_state.json seed data goes directly into InMemoryBridge._tasks/_projects/_tags. The _to_raw_format conversion runs on get_all OUTPUT. But the internal _handle_add_task/_handle_edit_task methods expect model-format internal state (they set "availability", not "status").

    This means we need the seed data to be in MODEL format (what InMemoryBridge expects internally), NOT raw format. But the initial_state.json from the capture script will be raw format.

    **Solution:** Add a `_from_raw_seed` method to convert raw seed data to model format, OR do the conversion in the contract test before seeding. The cleanest approach: in `_seed_bridge`, apply `adapt_snapshot` to the initial state before seeding:

    ```python
    def _seed_bridge(initial_state: dict[str, Any]) -> InMemoryBridge:
        """Seed InMemoryBridge from initial state (raw format from capture script).

        Applies adapt_snapshot to convert raw format to model format,
        since InMemoryBridge stores model format internally.
        """
        import copy
        from omnifocus_operator.bridge.adapter import adapt_snapshot

        adapted = copy.deepcopy(initial_state)
        adapt_snapshot(adapted)
        return InMemoryBridge(data={
            "tasks": list(adapted.get("tasks", [])),
            "projects": list(adapted.get("projects", [])),
            "tags": list(adapted.get("tags", [])),
            "folders": [],
            "perspectives": [],
        })
    ```
