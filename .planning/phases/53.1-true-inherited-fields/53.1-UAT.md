## Gap: Per-Field Aggregation Semantics (INHERIT-05 through INHERIT-10)

### Problem

`_walk_one` in `service/domain.py` uses **min** for all date fields. This is only correct for `due_date`. Three other aggregation strategies are needed, confirmed by empirical testing across 4 UAT sessions.

### Root cause

The original implementation (Plan 02/03) used a single comparison path for all non-flagged fields:

```python
# Current code (domain.py lines 247-250):
if cur is None or val < cur:
    ancestor_vals[inherited] = val
```

This applies `min` to every date field. The empirical study (FINDINGS.md) proved each field uses a different strategy.

### Per-field status

| Field | Current code | Correct rule | Status | Requirement |
|-------|-------------|-------------|--------|-------------|
| `inherited_due_date` | min | **min** — tightest deadline wins | CORRECT | INHERIT-05 |
| `inherited_defer_date` | min | **max** — latest block wins | BUG | INHERIT-06 |
| `inherited_planned_date` | min | **first-found** — nearest ancestor wins | BUG | INHERIT-07 |
| `inherited_flagged` | any-True | **any-True** — boolean OR | CORRECT | INHERIT-08 |
| `inherited_drop_date` | min | **first-found** — nearest ancestor wins | BUG | INHERIT-09 |
| `inherited_completion_date` | min | **first-found** — nearest ancestor wins | BUG | INHERIT-10 |

### Three semantic families

Discovered through empirical study — documented in `docs/architecture.md` and `.research/deep-dives/omnifocus-inheritance-semantics/FINDINGS.md`:

- **Constraint (min/max)**: Due and defer are constraints. Due = "must finish by X" (min — tightest wins). Defer = "can't start until X" (max — latest wins).
- **Override (first-found)**: Planned, drop, completion are events/intentions. Walk up, stop at first non-null. Own value always takes precedence; if unset, nearest ancestor wins. No aggregation across multiple ancestors.
- **Boolean OR**: Flagged is a signal. Any ancestor flagged → all descendants flagged.

### What the fix needs to do

In `_walk_one` (domain.py), replace the single `val < cur` comparison with field-specific logic:

1. **due_date** → keep `val < cur` (min) — already correct
2. **defer_date** → change to `val > cur` (max)
3. **planned_date, drop_date, completion_date** → take first non-null encountered in the walk, skip if already found (first-found)
4. **flagged** → already correct (any-True, separate code path)

Both loops need this change: the task ancestor walk (lines 241-250) and the project check (lines 257-267).

Also: remove FIXME comments at lines 229-231 after fix.
