## Tests

### 1. Project output has no inherited fields
expected: Call `get_project` on any project. The response should contain NO `inheritedFlagged`, `inheritedDueDate`, `inheritedDeferDate`, `inheritedPlannedDate`, `inheritedDropDate`, or `inheritedCompletionDate` fields. Projects can no longer have inherited fields — they were structurally removed.
result: pass

### 2. Inbox task: self-echoed inherited fields stripped
expected: Call `get_task` on a task that sits in the Inbox (no parent task, no project). Even if OmniFocus internally echoes values like `inheritedFlagged: false` or `inheritedDueDate` matching the task's own `dueDate`, these self-echoed values should NOT appear in the response.
result: pass

### 3. True inheritance from project: flagged
expected: Find or create a task inside a flagged project (the task itself is NOT flagged). Call `get_task` on that task. The response SHOULD include `inheritedFlagged: true` — this is a truly inherited value from the containing project.
result: pass

### 4. True inheritance from parent task: dates
expected: Find a subtask whose parent task has a due date, but the subtask itself does NOT have its own due date set. Call `get_task` on the subtask. The response SHOULD include `inheritedDueDate` matching the parent's due date — this is truly inherited.
result: pass

### 5. get_all applies true inheritance to tasks
expected: Call `get_all`. Examine tasks in the response. Tasks should reflect true inheritance: self-echoed inherited fields stripped, only truly inherited values present. Projects in the response should have NO inherited fields at all.
result: pass

### 6. list_tasks applies true inheritance
expected: Call `list_tasks` (with any filter). Tasks in the response should reflect true inheritance with correct per-field aggregation: min (due), max (defer), first-found (planned/drop/completion), any-True (flagged). Self-echoed values stripped, truly inherited values present.
result: pass
