# Phase 53.1 Plan 01: Move Inherited Fields from ActionableEntity to Task

Moved 5 inherited fields from ActionableEntity to Task, making it structurally impossible for projects to have inherited fields. Full 10-file cascade across models, repository mappers, config, descriptions, and tests.

## What Changed

### Model surgery (2 files)
- **common.py**: Removed `inherited_flagged`, `inherited_due_date`, `inherited_defer_date`, `inherited_planned_date`, `inherited_drop_date` from `ActionableEntity` and their 5 description imports
- **task.py**: Added all 5 fields alongside existing `inherited_completion_date` under `# Inherited fields (task-only -- projects cannot inherit)` comment; `inherited_flagged` gets `default=False`

### Repository mappers (3 files)
- **hybrid.py**: Removed 5 inherited key assignments from `_map_project_row`
- **adapter.py**: Added 5 `effective*` fields to `_PROJECT_EXTRA_DEAD_FIELDS`; stopped calling `_rename_inherited_fields` on projects in `adapt_snapshot`
- **bridge_only.py**: Fixed project flagged filter to use `p.flagged` (not `p.inherited_flagged`); created project-specific date field map using direct fields

### Config & descriptions (2 files)
- **config.py**: Removed 4 inherited fields from `PROJECT_DEFAULT_FIELDS` (13 -> 9 fields); removed `inheritedDropDate` from `PROJECT_FIELD_GROUPS["metadata"]`
- **descriptions.py**: Changed `_INHERITED_FIELD_DESC` from "entity" to "task"; removed `_INHERITED_PROJECTS_EXPLANATION`; updated `GET_PROJECT_TOOL_DOC` and `LIST_PROJECTS_TOOL_DOC` to remove inherited field references

### Tests (4 files)
- **test_inherited_field_migration.py**: New test file with 6 tests covering model structure, serialization, and adapter behavior
- **conftest.py**: `make_model_project_dict` reduced from 28 to 23 fields (no inherited keys)
- **test_models.py**: Updated ActionableEntity tests to not pass `inherited_flagged`; updated project field count assertions (28 -> 23)
- **test_list_pipelines.py**: Updated project date filtering fixture to use `dueDate` instead of `effectiveDueDate`
