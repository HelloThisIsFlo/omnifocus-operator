# Phase 53.1 Plan 02: Compute True Inheritance Walk + Service Wiring

Hierarchy walk on DomainLogic determines truly inherited fields by walking parent task chain + containing project, wired into all 3 task-returning service methods. Self-echoed inherited values reset to False/None, stripped by existing server layer.

## What Changed

### Domain logic (1 file)
- **domain.py**: Added `_INHERITED_FIELD_PAIRS` constant (6 field pairs), `compute_true_inheritance` async method, `_walk_one` private helper. Walk builds task_map + project_map from `get_all()`, iterates ancestor task chain via `parent.task.id`, checks containing project via `task.project.id` (Pitfall 3), resets unconfirmed inherited fields via `model_copy(update=...)`.

### Service wiring (1 file)
- **service.py**: `get_all_data()` walks raw.tasks then returns `raw.model_copy(update={"tasks": walked})`. `get_task()` wraps single task in list, walks, returns `walked[0]`. `_ListTasksPipeline._delegate()` walks `repo_result.items` before `_result_from_repo()`. `get_project()` unchanged (projects have no inherited fields).

### Tests (4 files)
- **test_service_domain.py**: 11 new tests in `TestComputeTrueInheritance` -- self-echo stripping (flagged, due_date), true inheritance from parent task, from project (D-01), deep hierarchy (grandparent), all 6 field pairs, inbox tasks, empty list, orphan task.
- **test_service.py**: 2 new integration tests verifying `get_all_data` and `get_task` strip self-echoed inherited fields through the full service chain.
- **test_list_pipelines.py**: 1 new integration test in `TestListTasksTrueInheritance` verifying `list_tasks` applies walk in pipeline.
- **test_server.py**: Fixed 2 camelCase naming tests that expected `inheritedFlagged` to always be present in output -- after walk, self-echoed `False` values are stripped.
