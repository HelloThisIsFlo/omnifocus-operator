  <name>Task 2: Catch Pydantic ValidationError noise in server.py</name>
  <files>src/omnifocus_operator/server.py, tests/test_server.py</files>
  <action>
  In server.py, the two `model_validate()` calls (add_tasks line 181, edit_tasks line 232) let Pydantic's ValidationError propagate raw, leaking noisy metadata (type=value_error, input_value, input_type, pydantic.dev URL).

  Wrap both model_validate calls in try/except:

  ```python
  from pydantic import ValidationError

  # In add_tasks (around line 181):
  try:
      spec = TaskCreateSpec.model_validate(items[0])
  except ValidationError as exc:
      messages = "; ".join(e["msg"] for e in exc.errors())
      raise ValueError(messages) from None

  # In edit_tasks (around line 232):
  try:
      spec = TaskEditSpec.model_validate(items[0])
  except ValidationError as exc:
      messages = "; ".join(e["msg"] for e in exc.errors())
      raise ValueError(messages) from None
  ```
