Add two new private methods to InMemoryBridge:

1. `_compute_effective_field(self, task: dict[str, Any], field: str) -> Any`:
   - Walk ancestor chain: check task's own value first, then follow `task.get("parent")` through task_index and project_index
   - Build task_index as `{t["id"]: t for t in self._tasks}` and project_index as `{p["id"]: p for p in self._projects}`
   - Use visited set for cycle protection
   - At each step: if current_id is a project, return `project_index[current_id].get(field)`. If a task, check `task.get(field)` -- return if non-null, else follow parent.
   - Return None if chain exhausted

2. `_compute_effective_flagged(self, task: dict[str, Any]) -> bool`:
   - Boolean OR semantics: true if task or ANY ancestor is flagged
   - Same chain walk as above but checks `entity.get("flagged", False)` -- returns True on first True, otherwise continues

Place both methods in the "Helpers" section (after `_recheck_has_children`, before "Operation handlers").
