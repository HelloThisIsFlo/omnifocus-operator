<objective>
Fix bridge-only mode returning project root tasks as regular task entries.

Purpose: In OmniFocus, every project has an underlying Task object. The SQL path excludes these via `LEFT JOIN ProjectInfo WHERE pi.task IS NULL`. The bridge-only path has no equivalent filter, causing `list_tasks` and `get_all` to include phantom task entries with self-referencing parents (parent.project.id === task.id). This is a pre-existing bug surfaced during Phase 43 UAT.

Output: Patched `adapt_snapshot` that filters project root tasks, plus test coverage.
</objective>
