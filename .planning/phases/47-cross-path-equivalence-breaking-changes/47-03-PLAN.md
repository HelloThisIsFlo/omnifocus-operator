<objective>
Fix `availability=[]` semantics in both repository paths so that an empty availability list matches zero remaining tasks instead of bypassing the filter entirely.

Purpose: UAT gap 2 found that `availability=[]` returns all remaining tasks (2958 in production) instead of 0. Both repo paths skip the filter for empty lists. This breaks the semantic contract that `availability=[]` + `completed: "all"` should return only completed tasks without any remaining tasks mixed in.

Output: Fixed bridge and SQL repo paths, updated repo-level tests, new cross-path equivalence test.
</objective>
