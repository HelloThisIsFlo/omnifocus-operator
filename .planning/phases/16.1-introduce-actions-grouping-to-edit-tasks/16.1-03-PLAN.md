<!-- New JSON shape as seen by MCP clients -->

```json
{
  "id": "task-id",
  "name": "Optional new name",
  "actions": {
    "tags": { "add": ["Work"], "remove": ["Old"] },
    "move": { "ending": "project-id" },
    "lifecycle": "complete"
  }
}
```

Field mapping from old to new JSON:
- "tags": ["a"] -> "actions": {"tags": {"replace": ["a"]}}
- "addTags": ["a"] -> "actions": {"tags": {"add": ["a"]}}
- "removeTags": ["a"] -> "actions": {"tags": {"remove": ["a"]}}
- "addTags": ["a"], "removeTags": ["b"] -> "actions": {"tags": {"add": ["a"], "remove": ["b"]}}
- "moveTo": {"ending": "x"} -> "actions": {"move": {"ending": "x"}}
</interfaces>

"""Edit existing tasks in OmniFocus using patch semantics.

Accepts an array of edit objects. Currently limited to 1 item per call.

Each item requires:
- id (required): Task ID to edit

Optional fields (omit to leave unchanged, set null to clear):

Actions block (omit to skip, groups stateful operations):

Returns array of results: [{success, id, name, warnings?}]
