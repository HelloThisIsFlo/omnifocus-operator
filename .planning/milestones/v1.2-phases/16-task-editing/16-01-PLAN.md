In `bridge.js`, add `handleEditTask(params)` function BEFORE the dispatch function, following the handleAddTask pattern:

1. **Task lookup**: `var task = Task.byIdentifier(params.id)` -- throw if not found
2. **Field updates** using `hasOwnProperty()` for ALL fields (falsy-safe):
   - `name`: `task.name = params.name`
   - `note`: `task.note = params.note` (can be null to clear)
   - `dueDate`: `task.dueDate = params.dueDate ? new Date(params.dueDate) : null`
   - `deferDate`: same pattern as dueDate
   - `plannedDate`: same pattern as dueDate
   - `flagged`: `task.flagged = params.flagged`
   - `estimatedMinutes`: `task.estimatedMinutes = params.estimatedMinutes` (can be null to clear)
3. **Tag management** -- check `params.hasOwnProperty("tagMode")`:
   - Resolve tagIds to Tag objects: `params.tagIds.map(id => Tag.byIdentifier(id))`, throw if any not found
   - `"replace"`: `task.clearTags(); if (tagObjs.length > 0) task.addTags(tagObjs);`
   - `"add"`: `task.addTags(tagObjs);`
   - `"remove"`: `task.removeTags(tagObjs);`
   - For `"add_remove"` mode (when both add and remove are requested): service layer sends `{tagMode: "add_remove", addTagIds: [...], removeTagIds: [...]}`. Process removals first, then additions.
4. **Movement** -- check `params.hasOwnProperty("moveTo")`:
   - `moveTo` has shape: `{position: "beginning"|"ending"|"before"|"after", containerId: "<id>"|null, anchorId: "<taskId>"}`
   - For `beginning`/`ending`: if `containerId` is null, use `inbox[position]`. Otherwise resolve container via `Project.byIdentifier(id) || Task.byIdentifier(id)`, throw if not found, use `container[position]`.
   - For `before`/`after`: resolve anchor via `Task.byIdentifier(anchorId)`, throw if not found, use `anchor[position]`.
   - Call `moveTasks([task], location)`
5. **Return**: `{ id: task.id.primaryKey, name: task.name }`

In dispatch function, add `else if (operation === "edit_task")` branch routing to `handleEditTask(request.params)`.
