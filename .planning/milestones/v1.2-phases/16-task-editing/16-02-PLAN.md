In `service.py`, add `edit_task` method on OperatorService (after add_task). Add TYPE_CHECKING imports for TaskEditSpec, TaskEditResult, MoveToSpec, UNSET.

**edit_task(self, spec: TaskEditSpec) -> TaskEditResult:**

1. **Verify task exists**: `task = await self._repository.get_task(spec.id)`. If None, raise ValueError.

2. **Validate name** (if not UNSET): reject empty/whitespace.

3. **Build payload** dict with only non-UNSET fields:
   - Start with `{"id": spec.id}`
   - For each field (name, note, due_date, defer_date, planned_date, flagged, estimated_minutes): if not UNSET, add to payload using camelCase key. For dates: serialize to ISO string if not None, else None.
   - CRITICAL: Do NOT use `model_dump(exclude_none=True)` -- that strips intentional nulls. Build payload manually, skipping only UNSET fields.

4. **Handle tags** (if any tag field is not UNSET):
   - Tag mutual exclusivity already validated by model_validator
   - If `spec.tags` is not UNSET (replace mode):
     - Resolve via `_resolve_tags(spec.tags)` (empty list = clear all)
     - Add to payload: `{"tagMode": "replace", "tagIds": resolved_ids}`
   - If `spec.add_tags` is not UNSET AND `spec.remove_tags` is not UNSET:
     - Resolve both via `_resolve_tags`
     - Add: `{"tagMode": "add_remove", "addTagIds": add_ids, "removeTagIds": remove_ids}`
   - If only `spec.add_tags` is not UNSET:
     - Resolve via `_resolve_tags`, add: `{"tagMode": "add", "tagIds": add_ids}`
   - If only `spec.remove_tags` is not UNSET:
     - Resolve via `_resolve_tags`, add: `{"tagMode": "remove", "tagIds": remove_ids}`
   - Collect warnings: for remove mode, check which tags the task currently has. For each tag in remove_tags that's not currently on the task, add warning: "Tag '{name}' was not on this task -- to skip tag changes, omit removeTags"

5. **Handle moveTo** (if not UNSET):
   - Extract the one set key from MoveToSpec (already validated by model_validator)
   - For `beginning`/`ending`:
     - If value is None: `moveTo = {"position": key, "containerId": None}` (inbox)
     - If value is a string: resolve via `_resolve_parent(value)`, then `moveTo = {"position": key, "containerId": value}`
   - For `before`/`after`:
     - Value is a task ID (anchor). Validate anchor exists via `get_task()`, raise ValueError if not found.
     - `moveTo = {"position": key, "anchorId": value}`
   - **Cycle detection** (only when moving to a container, not before/after):
     - For `beginning`/`ending` with a non-null containerId: check if containerId resolves to a task (not a project). If it's a task, walk parent chain from containerId upward. If we encounter `spec.id`, raise ValueError "Cannot move task: would create circular reference".
     - Use `get_all()` to get task data, build a task_map, walk `parent.id` upward while `parent.type == "task"`.
   - Add `{"moveTo": moveTo_dict}` to payload

6. **Delegate to repository**: `result = await self._repository.edit_task(payload)`

7. **Return TaskEditResult**: `TaskEditResult(success=True, id=result["id"], name=result["name"], warnings=warnings or None)`
