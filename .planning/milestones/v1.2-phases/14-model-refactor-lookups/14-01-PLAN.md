    2. **Task model** (`models/task.py`): Remove `project: str | None = None` and `parent: str | None = None`. Add `parent: ParentRef | None = None`. Add TYPE_CHECKING import for ParentRef. Update docstring.

    3. **models/__init__.py**: Add `ParentRef` to imports, add to `_ns` dict, add `ParentRef.model_rebuild(_types_namespace=_ns)` BEFORE `Task.model_rebuild()`. Add to `__all__`.

    4. **Bridge adapter** (`adapter.py`): In `_adapt_task`, after status mapping, transform the bridge's `project` and `parent` string fields into the new `parent` dict shape:
       - If `raw.get("parent")` is not None (has parent task): set `raw["parent"] = {"type": "task", "id": raw["parent"], "name": raw.get("parentName", "")}` and remove `raw.pop("project", None)`.
       - Elif `raw.get("project")` is not None (in project): set `raw["parent"] = {"type": "project", "id": raw["project"], "name": raw.get("projectName", "")}` and remove old `raw.pop("project")`.
       - Else: set `raw["parent"] = None` and remove `raw.pop("project", None)`.
       NOTE: The bridge sends `project` as the project ID and `parent` as the parent task ID (both strings). The bridge also sends `projectName` and `parentName` convenience fields -- check bridge.js to confirm field names. If bridge doesn't send name fields, use empty string.

    5. **HybridRepository** (`hybrid.py`): Update `_map_task_row` to build ParentRef dict. The key challenge:
       - `row["containingProjectInfo"]` is `ProjectInfo.pk` (NOT persistentIdentifier). Must JOIN to get project ID.
       - `row["parent"]` is the parent task's `persistentIdentifier` (direct ID, but no name).

       In `_read_all()`:
       - Build a `project_info_lookup: dict[str, dict[str, str]]` mapping `ProjectInfo.pk -> {"id": project_persistentIdentifier, "name": project_name}`. Query: `SELECT pi.pk, pi.task, t.name FROM ProjectInfo pi JOIN Task t ON pi.task = t.persistentIdentifier`.
       - Build a `task_name_lookup: dict[str, str]` mapping `persistentIdentifier -> name` from all Task rows (query: `SELECT persistentIdentifier, name FROM Task`).
       - Pass both lookups to `_map_task_row`.

       In `_map_task_row`, replace `"project": row["containingProjectInfo"], "parent": row["parent"]` with:
       ```python
       "parent": _build_parent_ref(row, project_info_lookup, task_name_lookup),
       ```

       Add helper `_build_parent_ref(row, project_info_lookup, task_name_lookup)`:
       - If `row["parent"]` is not None: return `{"type": "task", "id": row["parent"], "name": task_name_lookup.get(row["parent"], "")}`
       - Elif `row["containingProjectInfo"]` is not None: look up in project_info_lookup, return `{"type": "project", "id": info["id"], "name": info["name"]}`
       - Else: return None
