1. Add `_compute_tag_diff` method to OperatorService:
   - Signature: `async def _compute_tag_diff(self, tag_actions: TagActionSpec, current_tags: list[TagRef]) -> tuple[list[str], list[str], list[str]]`
   - Returns `(add_ids, remove_ids, warnings)`
   - Resolve tags via `_resolve_tags` inside the helper (keeps it self-contained)
   - Build `current_ids = {t.id for t in current_tags}`
   - For replace mode: resolve IDs, `final = set(resolved_ids)`. `replace: null` or `replace: []` means `final = set()`. If diff empty, warning "Tags already match the requested set"
   - For add mode: resolve IDs, `final = current_ids | set(resolved_ids)`. Per-tag warning if ID already in current_ids
   - For remove mode: resolve IDs, `final = current_ids - set(resolved_ids)`. Per-tag warning if ID not in current_ids
   - For add+remove: resolve both, `final = (current_ids | set(add_ids)) - set(remove_ids)`. Per-tag warnings for add duplicates and remove absents
   - CRITICAL (Pitfall 6): Generate warnings by comparing user input against current state BEFORE computing final set. Warnings reflect user intent, not diff result.
   - Compute: `to_add = list(final - current_ids)`, `to_remove = list(current_ids - final)`
   - For warning display names: fetch `all_tag_names` dict from `get_all()` (same pattern as current code)

   - Extend warnings with returned warnings
