    - Ambiguous tag name (multiple matches) raises ValueError listing matching IDs
    - Non-existent tag name tries ID fallback via get_tag. If still not found, raises ValueError
    - service.add_task with no parent -> parent stays None (inbox)
    - service.add_task validates name is not empty string, raises ValueError if so
    - service.add_task returns TaskCreateResult from repository
    - All validation happens BEFORE repository.add_task is called
  </behavior>

    **Service changes:**
    Add to OperatorService:
    - `async def add_task(self, spec: TaskCreateSpec) -> TaskCreateResult`:
      - Validate name: `if not spec.name or not spec.name.strip(): raise ValueError("Task name is required")`
      - Resolve parent if provided: call `_resolve_parent(spec.parent)`
      - Resolve tags if provided: call `_resolve_tags(spec.tags)` -> returns list of tag IDs
      - Call `self._repository.add_task(spec, resolved_tag_ids=resolved_tag_ids)`
      - Return result

    - `async def _resolve_parent(self, parent_id: str) -> str`:
      - Try `self._repository.get_project(parent_id)` -- if found, return parent_id
      - Try `self._repository.get_task(parent_id)` -- if found, return parent_id
      - Raise `ValueError(f"Parent not found: {parent_id}")` per user decision

      - For each name in tag_names:

        - Exactly 1 match -> use its ID
        - Multiple matches -> raise ValueError with ambiguous message including tag IDs (per user decision: show parent path if available)
        - No match -> try as ID fallback: `self._repository.get_tag(name)`. If found, use its ID. If not, raise ValueError
      - Return list of resolved tag IDs
