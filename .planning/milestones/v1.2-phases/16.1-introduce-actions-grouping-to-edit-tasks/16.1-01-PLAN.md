  <name>Task 2: Refactor service edit_task to read from spec.actions</name>
  <files>src/omnifocus_operator/service.py</files>
  <action>
Refactor `edit_task()` in `OperatorService` to read tags/move/lifecycle from `spec.actions` instead of top-level fields:

1. **Lifecycle fail-fast** (FIRST check after task existence + name validation):
   - If `spec.actions` is not UNSET AND `spec.actions.lifecycle` is not UNSET:
   - Raise `ValueError("Lifecycle actions are not yet implemented (coming in Phase 17)")`

2. **Tags handling** -- replace the current `has_replace`/`has_add`/`has_remove` block (lines ~174-234):
   - If `spec.actions` is UNSET or `spec.actions.tags` is UNSET -> skip tags entirely
   - Otherwise, read from `spec.actions.tags` (a `TagActionSpec` instance):
     - `has_replace = not isinstance(tag_actions.replace, _Unset)`
     - `has_add = not isinstance(tag_actions.add, _Unset)`
     - `has_remove = not isinstance(tag_actions.remove, _Unset)`
   - Same bridge payload translation as before:
     - `replace` -> `tagMode="replace"`, `tagIds=[resolved]` (replace: None means clear all = empty list)
     - `add` + `remove` -> `tagMode="add_remove"`, `addTagIds=[...]`, `removeTagIds=[...]`
     - `add` only -> `tagMode="add"`, `tagIds=[resolved]`
     - `remove` only -> `tagMode="remove"`, `removeTagIds=[resolved]`
   - Same warning logic for duplicate adds and absent removes (reuse existing patterns)

3. **Move handling** -- replace current `spec.move_to` block (lines ~237-268):
   - If `spec.actions` is UNSET or `spec.actions.move` is UNSET -> skip move
   - Otherwise, `move_to_spec = spec.actions.move` (same MoveToSpec type)
   - Rest of the logic is identical (resolve container, cycle detection, anchor validation)

4. **Empty actions block detection**:
   - If `spec.actions` is not UNSET but all three sub-fields (tags, move, lifecycle) are UNSET:
   - This is equivalent to an empty actions block `{}`
   - Should be treated same as "no actions specified" -- the existing `len(payload) == 1` check handles this naturally since no tag/move payload keys get added

5. **No-op detection** stays the same structurally -- still checks `tagMode` in payload and `moveTo` in payload. No changes needed there.

6. Update imports: add `ActionsSpec`, `TagActionSpec` to the TYPE_CHECKING import block from `omnifocus_operator.models.write`

Important: do NOT change the bridge payload format. The service still outputs `tagMode`, `tagIds`, `addTagIds`, `removeTagIds`, `moveTo` dict. Only the SOURCE of the data changes (from spec.actions.* instead of spec.tags/spec.add_tags/etc).
