<objective>
Finalise the read-surface strip and hierarchy-include rules so agents receive the new field shape. Remove the pre-existing defensive `availability` entry from `NEVER_STRIP` (STRIP-11), add `completesWithChildren` to `NEVER_STRIP` so `false` survives default stripping (PROP-08), and expand the `hierarchy` include group on both tasks and projects to carry `type`, `completesWithChildren`, and `hasChildren` (HIER-01/HIER-02). Add the contract test proving the no-suppression invariant (FLAG-06 / HIER-04): when both the default flags AND the `hierarchy` group apply, BOTH sets emit — redundancy is the requirement.

Purpose: projection is the single-responsibility boundary for what agents see vs internal state. This plan adjusts that boundary precisely, without touching upstream data flow. The default-response strip-when-false semantics already work for all five new derived flags (isSequential, dependsOnChildren, hasNote, hasRepetition, hasAttachments) because `strip_entity` already strips `False`/`""`/`None`/empty-list values — no new logic needed there. The only additions are `NEVER_STRIP` shuffling + field-group expansion.

Output:
- `NEVER_STRIP` contains `completesWithChildren` (added) and NOT `availability` (removed).
- `TASK_FIELD_GROUPS["hierarchy"]` and `PROJECT_FIELD_GROUPS["hierarchy"]` include `type` and `completesWithChildren`.
- `hasChildren` name preserved.
- Contract test proves no-suppression invariant for a `(type=sequential, has_children=True, completes_with_children=False)` task when `include=["hierarchy"]` is requested.
</objective>
