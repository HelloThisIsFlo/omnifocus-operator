<objective>
Surface the new read shape to agents through tool descriptions and model field docs (FLAG-07) AND prove derived read-only flags are rejected on write inputs (FLAG-08). The projection + model layers already handle the mechanics — this plan is the agent-facing communication contract.

Purpose: `dependsOnChildren` and `isSequential` are **load-bearing agent-behavior signals**, not cosmetic presence flags. If an agent reads them as generic "extra info" rather than as markers that change how it should reason about actionability, the milestone's primary value is lost. Tool descriptions must state the behavioral meaning explicitly and in the right terms.

`extra="forbid"` is already active on `AddTaskCommand` / `EditTaskCommand` via the `CommandModel` base class. This plan proves that via tests — generic Pydantic schema error is the LOCKED response (no custom educational message; JSON Schema already tells agents which fields are writable).
