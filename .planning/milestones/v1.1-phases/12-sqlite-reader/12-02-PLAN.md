  <behavior>
    - test_freshness_wal_polling: After TEMPORARY_simulate_write(), get_all() polls WAL mtime; when WAL mtime changes, read proceeds with fresh data
    - test_freshness_db_fallback: When WAL file does not exist, freshness uses main .db file mtime instead
    - test_freshness_timeout: If WAL mtime never changes within 2s, get_all() returns data anyway (no error)
    - test_freshness_poll_interval: Polling occurs at ~50ms intervals (verify via timing or mock)
    - test_freshness_no_stale_flag_normal_read: Normal get_all() (without simulate_write) does NOT trigger freshness polling -- reads immediately
    - test_simulate_write_marks_stale: TEMPORARY_simulate_write() sets internal stale flag; next get_all() clears it
    - test_simulate_write_not_on_protocol: TEMPORARY_simulate_write is on HybridRepository only, not on Repository protocol
  </behavior>
  <action>
    Add freshness state to HybridRepository.__init__:
    - `self._stale: bool = False` -- set by TEMPORARY_simulate_write(), cleared after fresh read
    - `self._last_wal_mtime_ns: int = 0` -- tracks WAL mtime for change detection

    Add `TEMPORARY_simulate_write(self) -> None`:
    - Captures current WAL (or DB) mtime as baseline
    - Sets `self._stale = True`
    - Comment: "Delete this method when real writes are implemented"
    - This method is NOT async (just sets a flag and reads mtime synchronously is fine, or make it async with to_thread for os.stat)

    Modify `get_all()`:
    - If `self._stale` is True, call `await self._wait_for_fresh_data()` first, then clear stale flag
    - If not stale, read immediately (no polling)

    Add `async def _wait_for_fresh_data(self) -> None`:
    - WAL path: `self._db_path.parent / (self._db_path.name + "-wal")`
    - Poll loop: `time.monotonic()` deadline = now + 2.0 seconds
    - Each iteration:
      - Try `os.stat(wal_path).st_mtime_ns` via `asyncio.to_thread`
      - On FileNotFoundError: fall back to `os.stat(self._db_path).st_mtime_ns`
      - If current mtime != baseline (`self._last_wal_mtime_ns`): return (fresh data available)
      - Else: `await asyncio.sleep(0.05)` (50ms poll)
    - On timeout: return anyway (proceed with potentially stale data -- better than error)

    After freshness wait (or on normal non-stale reads), update `self._last_wal_mtime_ns` with the current WAL/DB mtime.

    **Test approach for freshness:**
    - Use file-based SQLite in tmp_path
    - Create a WAL file manually (touch it) to simulate WAL presence
    - For polling tests: modify WAL mtime in a background task after a short delay
    - For timeout tests: never modify WAL mtime, verify get_all() completes within ~2.5s
    - For DB fallback: ensure no WAL file exists, modify DB mtime instead
    - Mock `asyncio.sleep` or use timing assertions for poll interval verification
  </action>
