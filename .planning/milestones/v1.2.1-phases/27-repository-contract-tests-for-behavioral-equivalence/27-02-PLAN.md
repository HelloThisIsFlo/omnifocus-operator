    This file loads golden master fixtures from `tests/golden/` and replays all scenarios against InMemoryBridge, verifying the output matches.

    **Imports:**
    ```python
    import json
    import pytest
    from pathlib import Path
    from tests.doubles import InMemoryBridge
    from tests.golden.normalize import (
        normalize_response,
        normalize_state,
        filter_to_known_ids,
    )
    ```

    **Constants:**
    ```python
    GOLDEN_DIR = Path(__file__).parent / "golden"
    INITIAL_STATE_FILE = GOLDEN_DIR / "initial_state.json"
    ```

    **Helper functions:**

    `load_initial_state() -> dict` -- loads and returns `initial_state.json`. Raises `pytest.skip("Golden master not captured yet. Run: uv run python uat/capture_golden_master.py")` if file doesn't exist.

    `load_scenarios() -> list[dict]` -- loads all `scenario_*.json` files from GOLDEN_DIR, sorted by filename. Returns list of scenario dicts. Raises `pytest.skip("No golden master scenarios found. Run: uv run python uat/capture_golden_master.py")` if no files exist.

    **Test class `TestBridgeContract`:**

    The test is sequential: scenarios build on accumulated state in InMemoryBridge, matching how the capture script recorded them.

    `test_all_scenarios_match_golden_master(self)`:
    1. Load initial state -> seed InMemoryBridge with it (projects, tags, empty tasks/folders/perspectives)
    2. Load all scenarios in order
    3. Maintain `known_task_ids`, `known_project_ids`, `known_tag_ids` sets (initialized from initial state)
    4. For each scenario:
       a. Execute `response = await bridge.send_command(scenario["operation"], scenario["params"])`
       b. If operation is `add_task`, add `response["id"]` to `known_task_ids`
       c. `actual_response = normalize_response(response)`
       d. `assert actual_response == scenario["response"]`, with message: f"Response mismatch in scenario {scenario['scenario']}"
       e. `state = await bridge.send_command("get_all")`
       f. `filtered = filter_to_known_ids(state, known_task_ids, known_project_ids, known_tag_ids)`
       g. `actual_state = normalize_state(filtered)`
       h. `assert actual_state == scenario["state_after"]`, with message: f"State mismatch in scenario {scenario['scenario']}"

    Use `@pytest.mark.asyncio` on the test method. The test is a single method (not parametrized) because state accumulates across scenarios -- parametrized tests would each need independent state.

    **Additional test: `test_golden_master_files_exist(self)`**
    Simple smoke test that verifies `initial_state.json` exists and at least one `scenario_*.json` exists. Uses `pytest.skip` if not present (so CI doesn't fail before first capture, but the test is visible as skipped).

    **Edge case handling:**
    - If golden master files don't exist (user hasn't run capture yet), tests skip with clear message: "Golden master not captured yet. Run: uv run python uat/capture_golden_master.py"
    - The skip ensures CI doesn't fail on a fresh clone, but the tests are visible as pending
