### Step 2: Migrate individual test sites

**Key: use `make_snapshot_dict()` not `make_snapshot()`** for bridge seeding. InMemoryBridge takes a dict, not a Pydantic model.

**Two migration patterns depending on test shape:**

**Pattern A -- Tests using default snapshot (majority):**
```python
# BEFORE:
async def test_something(self) -> None:
    snapshot = make_snapshot()
    repo = InMemoryRepository(snapshot=snapshot)
    service = OperatorService(repository=repo)
    ...

# AFTER -- inject repo fixture (per D-12):
async def test_something(self, repo: BridgeRepository) -> None:
    service = OperatorService(repository=repo)
    ...
```

**Pattern B -- Tests needing custom snapshot data:**
```python
# BEFORE:
async def test_custom(self) -> None:
    snapshot = make_snapshot(tasks=[custom_task], projects=[custom_project])
    repo = InMemoryRepository(snapshot=snapshot)
    ...

# AFTER -- construct inline following the fixture composition pattern (per D-11):
async def test_custom(self) -> None:
    bridge = InMemoryBridge(data=make_snapshot_dict(tasks=[custom_task_dict], projects=[custom_project_dict]))
    repo = BridgeRepository(bridge=bridge, mtime_source=ConstantMtimeSource())
    ...
```

For Pattern B, if many tests in a class share the same custom snapshot, create a class-level or scoped fixture instead of repeating inline.

**Pattern C -- Tests needing both repo and bridge access (per D-12):**
```python
# Inject both fixtures:
async def test_with_bridge_access(self, repo: BridgeRepository, bridge: InMemoryBridge) -> None:
    service = OperatorService(repository=repo)
    # Can inspect bridge.calls, bridge.call_count, etc.
    ...
```

### Critical pitfall -- ConstantMtimeSource caching behavior:
ConstantMtimeSource always returns 0. BridgeRepository starts with `_last_mtime_ns = 0`. On first get_all: `_cached is None` triggers refresh (good).
