       - If `on is not None` and `on_dates is not None`: raise `ValueError("on and on_dates are mutually exclusive on monthly frequency. Use on for day-of-week patterns (e.g., {'second': 'tuesday'}) or onDates for specific dates (e.g., [1, 15]).")`

       - Return normalized list

    7. Add `@field_validator("on", mode="before")` on `Frequency` (per D-13):
       - If value is None, return None
       - Lowercase and validate ordinal against `{"first", "second", "third", "fourth", "fifth", "last"}`
       - Lowercase and validate day name against `{"monday", "tuesday", "wednesday", "thursday", "friday", "saturday", "sunday", "weekday", "weekend_day"}`
       - Raise `ValueError` with `REPETITION_INVALID_ORDINAL` / `REPETITION_INVALID_DAY_NAME` on invalid values
