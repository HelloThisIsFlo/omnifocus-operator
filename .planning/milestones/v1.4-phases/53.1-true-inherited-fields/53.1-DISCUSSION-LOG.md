## Walk depth: project as final ancestor

| Option | Description | Selected |
|--------|-------------|----------|
| Task chain + project | Walk parent.task.id up, then check containing project as final ancestor. Builds project_map from same get_all() call. | ✓ |

**User's choice:** Task chain + project (Recommended)
**Notes:** Research confirmed that projects CAN have dates/flags that tasks inherit. Without checking the project, a task under a flagged project would have inheritedFlagged incorrectly stripped — the exact bug this phase exists to fix.

## Stripping mechanism for tasks

| Option | Description | Selected |
|--------|-------------|----------|
| Set to None, widen inherited_flagged to bool \| None | Semantically honest — None means "not inherited." Requires type change. | |
| Set False/None, keep current types | False for inherited_flagged (stripped by D-03), None for dates. No type changes. | ✓ |

**User's choice:** Keep current types — False for flagged, None for dates
**Notes:** User's reasoning: "boolean false is the same as date being null — it's there or not there. Instead of having a variable that has only one value and that is either there or none, what you use instead is a boolean." Claude concurred: OmniFocus flagging is an OR operation. A parent having flagged=False doesn't "contribute" anything. So inherited_flagged is binary: True (some ancestor flagged) or False (nobody did). No meaningful "ancestor explicitly set False" case.
