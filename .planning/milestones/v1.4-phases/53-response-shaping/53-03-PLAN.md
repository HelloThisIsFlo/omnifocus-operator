**server/projection.py** — Create the projection module (per D-03, D-07, D-09):

Key functions:
- `NEVER_STRIP: frozenset[str] = frozenset({"availability"})` — per STRIP-02
- `STRIP_VALUES: set = {None, [], "", False, "none"}` — per STRIP-01
- `strip_entity(entity: dict[str, Any]) -> dict[str, Any]` — remove keys whose values are in STRIP_VALUES, except NEVER_STRIP keys
- `strip_all_entities(data: dict[str, Any]) -> dict[str, Any]` — for get_all response, strip each entity in each collection (tasks, projects, tags, folders, perspectives)
- `resolve_fields(include, only, default_fields, field_groups) -> tuple[frozenset[str] | None, list[str]]` — resolve which fields to keep and return any warnings. Returns (allowed_fields_or_None, warnings). None means "keep all stripped fields" (no projection).
- `project_entity(entity: dict[str, Any], allowed_fields: frozenset[str]) -> dict[str, Any]` — keep only allowed_fields keys
- `shape_list_response(result: ListResult[T], include, only, default_fields, field_groups, warnings_from_service) -> dict[str, Any]` — full pipeline: serialize items, strip, project, reassemble envelope

The `shape_list_response` function:
1. Serialize each item via `item.model_dump(by_alias=True)`
2. Strip each entity dict via `strip_entity()`
3. If include or only specified, resolve fields and project
4. Build envelope: `{"items": [...], "total": result.total, "hasMore": result.has_more}`
5. If warnings exist (from service + from projection), add `"warnings": [...]`
6. Return the dict

Important implementation detail per D-07: strip THEN project. Clean the entity first, then select fields.

For `include`/`only` conflict handling (D-06): `only` takes precedence, `include` is ignored, add warning: "include and only are mutually exclusive. include was ignored because only was provided. Use one or the other."

For invalid `only` field names (FSEL-07): case-insensitive matching. Build a lowercase->original mapping from valid field names. Unrecognized names after case-insensitive lookup produce a warning per invalid name.

For `include: ["*"]` (FSEL-08): union of default_fields with all group fields.
