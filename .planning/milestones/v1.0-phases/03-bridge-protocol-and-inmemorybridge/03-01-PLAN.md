  <behavior>
    **Protocol (BRDG-01):**
    - Bridge protocol defines `async send_command(operation: str, params: dict[str, Any] | None = None) -> dict[str, Any]`
    - InMemoryBridge structurally satisfies Bridge protocol (assign `bridge: Bridge = InMemoryBridge(data={})` type-checks)
    - Protocol uses `...` body (not pass, not raise NotImplementedError)

    **Error hierarchy (BRDG-01):**
    - BridgeError is base, stores `operation: str`, accepts `cause: Exception | None` keyword arg
    - BridgeTimeoutError stores `timeout_seconds: float`, message format: `"Operation '{op}' timed out after {s}s"`
    - BridgeConnectionError stores `reason: str`, message format: `"Cannot connect to OmniFocus: {reason}"`
    - BridgeProtocolError stores `detail: str`, message format: `"Protocol error on '{op}': {detail}"`
    - All subclasses inherit from BridgeError (`issubclass` checks)
    - Exception chaining via `__cause__` works correctly

    **InMemoryBridge (BRDG-02):**
    - `InMemoryBridge(data={"key": "value"})` -- constructor injection of return data
    - `InMemoryBridge(data=None)` or `InMemoryBridge()` -- defaults to empty dict `{}`
    - `await bridge.send_command("dump_all")` returns the configured data dict
    - `await bridge.send_command("op", params={"k": "v"})` records params
    - `bridge.call_count` returns number of invocations
    - `bridge.calls` returns list of `BridgeCall(operation=..., params=...)` records
    - `bridge.calls` returns a copy (mutating it doesn't affect internal state)
    - `bridge.set_error(BridgeTimeoutError(...))` causes next `send_command` to raise that error
    - Call is recorded BEFORE error is raised (call_count increments even on error)
    - `bridge.clear_error()` removes the configured error, subsequent calls succeed
    - `BridgeCall` is a frozen dataclass (immutable)
  </behavior>
