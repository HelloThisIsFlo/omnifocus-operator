**.pre-commit-config.yaml**:
```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.15.4
    hooks:
      - id: ruff-check
        args: [--fix]
      - id: ruff-format
  - repo: local
    hooks:
      - id: mypy
        name: mypy
        entry: uv run mypy src/
        language: system
        types: [python]
        require_serial: true
        pass_filenames: false
```

IMPORTANT: ruff-check MUST come BEFORE ruff-format (check --fix can produce code needing reformatting). The mypy hook uses `language: system` + `uv run mypy src/` to ensure mypy sees all project dependencies (NOT mirrors-mypy which runs in isolation and silently ignores missing imports). Use `pass_filenames: false` on the mypy hook so it always checks the full `src/` directory (cross-module type safety).

**.github/workflows/ci.yml**:
```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v7
        with:
          enable-cache: true

      - name: Set up Python
        run: uv python install 3.12

      - name: Install dependencies
        run: uv sync --locked --dev

      - name: Lint
        run: uv run ruff check

      - name: Format check
        run: uv run ruff format --check

      - name: Type check
        run: uv run mypy src/

      - name: Test
        run: uv run pytest --cov-fail-under=80
```

CI triggers on push to main and on pull requests only. Single Python version (3.12). Uses `--locked` to ensure reproducible installs (fails if uv.lock is out of date). Steps run in order: lint -> format check -> type check -> test.
