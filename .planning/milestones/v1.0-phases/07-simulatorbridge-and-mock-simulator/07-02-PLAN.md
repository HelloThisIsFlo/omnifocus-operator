    Create `src/omnifocus_operator/simulator/__main__.py`:
    - Runnable via `python -m omnifocus_operator.simulator`
    - CLI args via argparse:
      - `--ipc-dir PATH` (required): directory to watch for IPC files
      - `--fail-mode {timeout,error,malformed}` (optional, default None): failure simulation mode
      - `--fail-after N` (optional, default None): first N requests succeed, then fail mode activates
      - `--delay SECONDS` (optional, default 0.0): delay before writing response
    - On startup:
      - Configure logging to stderr (DEBUG level, timestamped format)
      - Call `sys.stderr.reconfigure(line_buffering=True)` to prevent buffering (fixes readiness detection)
      - Create IPC directory if needed (`mkdir(parents=True, exist_ok=True)`)
      - Print config summary to stderr (ipc_dir, fail_mode, fail_after, delay)
      - Print "Ready: watching <ipc_dir>" to stderr (fixture readiness marker)
    - Main loop:
      - Poll IPC directory every 100ms (`time.sleep(0.1)`)
      - Match files using regex `^\d+_[0-9a-f-]+\.request\.json$` (ONLY .request.json, NOT .tmp)
      - Track seen files (set of filenames) to avoid reprocessing
      - For each new request file:
        - Increment request counter
        - Log "Request #N: filename" to stderr
        - Parse request JSON, extract `dispatch` field, split on `::::` to get uuid and operation
        - Extract PID from filename (split on `_`, take first part) -- use request PID, NOT simulator PID
        - If `--delay` > 0: `time.sleep(delay)` before writing response
        - Check if should fail: `--fail-mode` is set AND (no `--fail-after` OR request_count > fail_after)
        - For `--fail-mode timeout`: log "Simulating timeout" and do NOT write response file
        - For `--fail-mode error`: write `{"success": false, "error": "simulated error"}` atomically
        - For `--fail-mode malformed`: write `"not valid json {{{" ` atomically
        - For normal success: write `{"success": true, "data": SIMULATOR_SNAPSHOT}` atomically
        - Atomic write pattern: write to `.tmp`, then `os.replace()` to final `.response.json` path
        - Log "Response: filename" or "Timeout simulated" to stderr
    - Handle KeyboardInterrupt gracefully (log "Shutting down" and exit)
