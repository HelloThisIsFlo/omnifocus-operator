def _map_task_availability(*, blocked, date_completed, date_hidden):
    if date_hidden is not None: return "dropped"
    if date_completed is not None: return "completed"
    if blocked: return "blocked"
    return "available"

    if date_hidden is not None: return "dropped"
    if effective_status == "dropped": return "dropped"
    if date_completed is not None: return "completed"
    if effective_status == "inactive": return "blocked"

  <name>Parameterized SQL query builder</name>
  <files>src/omnifocus_operator/repository/query_builder.py, tests/test_query_builder.py</files>
  <behavior>
    **SqlQuery return type:**
    - `SqlQuery` is a NamedTuple with `sql: str` and `params: tuple[Any, ...]`
    - Each builder returns a pair: `(data_query: SqlQuery, count_query: SqlQuery)`

    **build_list_tasks_sql(query: ListTasksQuery) -> tuple[SqlQuery, SqlQuery]:**
    - Base SQL: `SELECT t.* FROM Task t LEFT JOIN ProjectInfo pi ON t.persistentIdentifier = pi.task WHERE pi.task IS NULL`
    - in_inbox=True -> `AND t.inInbox = ?` with param `1`; in_inbox=False -> param `0`
    - flagged=True -> `AND t.flagged = ?` with param `1`; flagged=False -> param `0`
    - project="Work" -> `AND t.containingProjectInfo IN (SELECT pi2.task FROM ProjectInfo pi2 JOIN Task t2 ON pi2.task = t2.persistentIdentifier WHERE t2.name LIKE ? COLLATE NOCASE)` with param `%Work%`
    - tags=["id1","id2"] -> `AND t.persistentIdentifier IN (SELECT ttg.task FROM TaskToTag ttg WHERE ttg.tag IN (?,?))` with params `("id1","id2")` (note: tags are already resolved to IDs by service per D-06b)
    - estimated_minutes_max=30 -> `AND t.estimatedMinutes <= ?` with param `30`
    - availability=["available","blocked"] -> `AND ((t.blocked = 0 AND t.dateCompleted IS NULL AND t.dateHidden IS NULL) OR (t.blocked != 0 AND t.dateCompleted IS NULL AND t.dateHidden IS NULL))`
    - availability=["completed"] -> `AND ((t.dateCompleted IS NOT NULL AND t.dateHidden IS NULL))`
    - availability=["dropped"] -> `AND ((t.dateHidden IS NOT NULL))`
    - search="foo" -> `AND (t.name LIKE ? COLLATE NOCASE OR t.plainTextNote LIKE ? COLLATE NOCASE)` with params `(%foo%, %foo%)`
    - limit=10 -> `LIMIT ?` with param `10`
    - offset=5 (only with limit) -> `OFFSET ?` with param `5`
    - limit=0 -> count query only (data query has `LIMIT 0`)
    - Count query: same WHERE clauses but `SELECT COUNT(*) FROM Task t LEFT JOIN ... WHERE ...` (no LIMIT/OFFSET)
    - All user-provided values use `?` placeholders (INFRA-01)
    - No filters (default query) -> only availability default applies

    **build_list_projects_sql(query: ListProjectsQuery) -> tuple[SqlQuery, SqlQuery]:**
    - Base SQL: `SELECT t.*, pi.lastReviewDate, pi.nextReviewDate, pi.reviewRepetitionString, pi.nextTask, pi.folder, pi.effectiveStatus FROM Task t JOIN ProjectInfo pi ON t.persistentIdentifier = pi.task`
    - availability=["available","blocked"] -> compound OR using project availability logic:
      - available: `(pi.effectiveStatus NOT IN ('dropped','inactive') AND t.dateCompleted IS NULL AND t.dateHidden IS NULL)`
      - blocked: `(pi.effectiveStatus = 'inactive' AND t.dateCompleted IS NULL AND t.dateHidden IS NULL)`
      - completed: `(t.dateCompleted IS NOT NULL AND t.dateHidden IS NULL AND pi.effectiveStatus != 'dropped')`
      - dropped: `(t.dateHidden IS NOT NULL OR pi.effectiveStatus = 'dropped')`
    - folder="Home" -> `AND pi.folder IN (SELECT f.persistentIdentifier FROM Folder f WHERE f.name LIKE ? COLLATE NOCASE)` with param `%Home%`
    - review_due_within="1w" -> service resolves to a datetime string, query builder receives it as-is in the field. The query builder adds `AND pi.nextReviewDate IS NOT NULL AND pi.nextReviewDate <= ?` with the resolved datetime param. NOTE: The service layer (Phase 37) resolves "1w" to a concrete datetime. The query builder just handles the raw comparison — but for Phase 34, the `review_due_within` field on the query model is a string. The query builder should NOT parse duration strings — it passes the field value through. Phase 37 service will resolve the duration to a timestamp and set a resolved field. For now, the query builder simply appends `AND pi.nextReviewDate IS NOT NULL AND pi.nextReviewDate <= ?` with the string value of `review_due_within` as the param. If `review_due_within` is None, no clause added.
    - flagged=True -> `AND t.flagged = ?` with param `1`
    - limit/offset -> same as tasks
    - Count query: `SELECT COUNT(*) FROM Task t JOIN ProjectInfo pi ON ...` with same WHERE clauses

Key implementation details:
- Build WHERE clauses as a `list[str]` of conditions, append params to a `list[Any]`
- Availability clause: helper function `_task_availability_clause(values)` and `_project_availability_clause(values)` that return `(clause_str, params_list)`. No params needed for availability (all conditions are column comparisons, no user values).
