    **Architecture** (per D-05a-e):

    Create `tests/test_cross_path_equivalence.py` with:

    1. **Neutral test data** -- define test entities once in a format that both seed adapters can translate. Use model-level dicts (matching the model field names). Include:
       - 3-4 tasks (mix of: inbox, flagged, different projects, different tags, different availability, with/without estimated_minutes)
       - 2-3 projects (mix of: different availability, different folders, with/without next_review_date, flagged/unflagged)
       - 2-3 tags (mix of: active, on_hold)
       - 2 folders (mix of: active, on_hold)
       - 1-2 perspectives (builtin and custom)

    2. **`seed_bridge_repo(data)` adapter** -- takes neutral data, translates to bridge format (camelCase, ISO dates, inline tags), creates `InMemoryBridge` with `make_snapshot_dict()` / `make_task_dict()` / `make_project_dict()` helpers from conftest, returns `BridgeRepository`. Key translations:
       - Model field names -> camelCase bridge field names
       - datetime -> ISO 8601 strings
       - Availability enum -> bridge string format
       - Tag assignments -> inline tag arrays on tasks

    3. **`seed_sqlite_repo(data, tmp_path)` adapter** -- takes neutral data, translates to SQLite format (CF epoch floats, int booleans, join tables), creates test DB via `create_test_db()` pattern from `test_hybrid_repository.py`, returns `HybridRepository`. Key translations:
       - datetime -> CF epoch float: `(dt - datetime(2001, 1, 1, tzinfo=UTC)).total_seconds()`
       - bool -> int (0/1)
       - Tag assignments -> join table rows
       - Project-task relationships -> parent column

    4. **Parametrized repo fixture** (per D-05a):

       ```python
       @pytest.fixture(params=["bridge", "sqlite"])
       async def cross_repo(request, tmp_path):
           data = build_neutral_test_data()
           if request.param == "bridge":
               return await seed_bridge_repo(data)
           else:
               return await seed_sqlite_repo(data, tmp_path)
       ```

    If `create_test_db()` is not importable from `test_hybrid_repository.py` (it's a module-level function, not in conftest), either:
    - Extract it to a shared test helper (e.g., `tests/helpers/sqlite_seeding.py`) and import from both files
    - Or import directly from `test_hybrid_repository` if the function is accessible

    Check `tests/conftest.py` for any existing shared helpers that may already cover this.

    5. **Comparison helper** -- sort items by `id` before comparison (per D-05d):

       ```python
       def assert_equivalent(result_a, result_b):
           items_a = sorted(result_a.items, key=lambda x: x.id)
           items_b = sorted(result_b.items, key=lambda x: x.id)
           assert items_a == items_b
           assert result_a.total == result_b.total
       ```
