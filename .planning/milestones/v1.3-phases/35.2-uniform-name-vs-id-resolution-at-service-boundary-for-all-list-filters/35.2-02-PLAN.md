    Add a method that implements the resolution cascade (steps 1-2). The method should:
    - Accept a filter value (string) and a list of entities (any object with `.id` and `.name` attributes)
    - Step 1 (ID match): If the value matches any entity's `.id` exactly, return `[value]`
    - Step 2 (Substring match): Case-insensitive substring check (`value.lower() in entity.name.lower()`). Return all matching entity IDs
    - Return empty list if no match at step 1 or 2

    Signature (discretion area -- recommended single generic method):
    ```python
    def resolve_filter(self, value: str, entities: Sequence[T]) -> list[str]:
    ```

    For tags specifically (per RESEARCH open question 3), each tag in the list should be resolved independently through the cascade:

    ```python
    def resolve_filter_list(self, values: list[str], entities: Sequence[T]) -> list[str]:

    ```
    This returns a flat list of all resolved IDs across all values (deduped). For values that don't resolve, they are simply not included -- the pipeline handles the warning.

    Also add a method that returns the unresolved values from a list (for the pipeline to generate warnings):
