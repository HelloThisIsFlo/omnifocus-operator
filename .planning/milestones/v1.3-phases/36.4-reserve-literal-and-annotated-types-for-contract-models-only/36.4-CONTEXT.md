# Phase 36.4: Reserve Literal and Annotated types for contract models only - Context

**Gathered:** 2026-04-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Establish and enforce: Literal and Annotated constraint types live on contract models (`contracts/`), core models (`models/`) use plain types. Runtime validators enforce correctness on core models — the type system doesn't over-promise to the static checker. Includes moving type alias definitions from `models/` to `contracts/`, updating core model field annotations to plain types, adding Annotated constraints to contract fields that currently rely on runtime-only validation, documenting the convention, and adding an enforcement test.

</domain>

<decisions>
## Implementation Decisions

### Type alias location
- **D-01:** Move all four Literal/Annotated type aliases (`FrequencyType`, `DayCode`, `OnDate`, `DayName`) from `models/repetition_rule.py` to `contracts/`. They exist solely for JSON Schema benefit and should live where they're consumed.
- **D-02:** After moving, replace `_VALID_DAY_NAMES = set(DayName.__args__)` in `models/repetition_rule.py` with a plain set literal (consistent with how `_VALID_DAY_CODES` is already a plain set). This keeps the dependency direction clean: `contracts/` → `models/`, never reversed.
- **D-03:** Keep all four as named aliases (not inlined). Each is used on 2+ contract fields; `DayName` is used on 6 fields in `OrdinalWeekdaySpec`.

### Core model field types
- **D-04:** `Frequency.type: FrequencyType` → `str`. Runtime `_validate_frequency_type` validator (in contracts) already enforces correctness.
- **D-05:** `OrdinalWeekday` fields (`first` through `last`, 6 fields): `DayName | None` → `str | None`. Runtime `normalize_day_name` validator already enforces correctness.
- **D-06:** No changes needed for `Frequency.on_days` (`list[str]`) or `Frequency.on_dates` (`list[int]`) — already plain types.

### New Annotated constraints on contract fields
- **D-07:** Add `Annotated[int, Field(ge=1)]` to `FrequencyAddSpec.interval` and `FrequencyEditSpec.interval` — currently validated at runtime only, schema doesn't advertise `minimum: 1` to agents.
- **D-08:** Add `Annotated[int, Field(ge=1)]` to `EndByOccurrences.occurrences` — same situation, runtime-validated but schema-invisible.

### Convention documentation and enforcement
- **D-09:** Add a section to `docs/model-taxonomy.md` documenting the rule: Literal and Annotated constraint types belong on contract model fields, core models use plain types with runtime validators.
- **D-10:** Add an AST enforcement test that scans `models/` class bodies for Literal/Annotated field annotations. Must be scoped to Pydantic model field annotations within class bodies — not module-level type alias definitions (which may legitimately remain if any are kept for validation purposes). Follows existing project pattern (cf. `test_descriptions.py`).

### Folded Todos
- **Reserve Literal and Annotated types for contract models only** (area: models) — this todo is the origin of Phase 36.4. Covers the exact same scope. Mark done when phase completes.

</decisions>
