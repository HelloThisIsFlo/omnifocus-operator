**Resolution table** (assume now = 2026-04-07T14:00:00, Tuesday, week_start=0/Monday):

**String shortcuts:**
- `"today"` on any field -> `(2026-04-07T00:00, 2026-04-08T00:00)`
- `"overdue"` on `due` -> `(None, 2026-04-07T14:00)` (due before now)
- `"soon"` on `due` with granularity=1, interval=172800 -> `(None, 2026-04-09T00:00)` (midnight_today + 2 days)
- `"soon"` on `due` with granularity=0, interval=86400 -> `(None, 2026-04-08T14:00)` (now + 24h)
- `"soon"` without config -> raises ValueError with educational message
- `"any"` on `completed`/`dropped` -> raises ValueError (not a date filter -- handled by pipeline, not resolver)

**Shorthand {this: unit}:**
- `{this: "d"}` -> `(2026-04-07T00:00, 2026-04-08T00:00)` (today)
- `{this: "w"}` week_start=monday -> `(2026-04-07T00:00, 2026-04-14T00:00)` (Mon-Mon, now is Tuesday)
- `{this: "w"}` week_start=sunday -> `(2026-04-06T00:00, 2026-04-13T00:00)` (Sun-Sun)
- `{this: "m"}` -> `(2026-04-01T00:00, 2026-05-01T00:00)` (calendar month boundaries)
- `{this: "y"}` -> `(2026-01-01T00:00, 2027-01-01T00:00)` (calendar year boundaries)

**Shorthand {last: duration}:**
- `{last: "3d"}` -> `(2026-04-04T00:00, 2026-04-07T14:00)` (3 days ago midnight through now)
- `{last: "1w"}` or `{last: "w"}` -> `(2026-03-31T00:00, 2026-04-07T14:00)` (7 days ago midnight through now)
- `{last: "1m"}` -> 30 days ago midnight through now (RESOLVE-07 naive)
- `{last: "1y"}` -> 365 days ago midnight through now (RESOLVE-07 naive)

**Shorthand {next: duration}:**
- `{next: "2d"}` -> `(2026-04-07T14:00, 2026-04-10T00:00)` (now through midnight 3 days from now = rest of today + 2 full days)
- `{next: "1w"}` -> `(2026-04-07T14:00, 2026-04-15T00:00)` (now through midnight 8 days from now)
- `{next: "1m"}` -> now through 31 days from now midnight (RESOLVE-07: 30d + rest of today)

**Absolute:**
- `{before: "2026-04-14"}` -> `(None, 2026-04-15T00:00)` (RESOLVE-08: date-only = start of next day)
- `{after: "2026-04-01"}` -> `(2026-04-01T00:00, None)` (RESOLVE-09: date-only = start of that day)
- `{before: "2026-04-14T18:00:00"}` -> `(None, 2026-04-14T18:00)` (datetime = exact)
- `{after: "2026-04-01T09:00:00"}` -> `(2026-04-01T09:00, None)` (datetime = exact)
- `{before: "now"}` -> `(None, 2026-04-07T14:00)` (now = the passed-in timestamp)
- `{after: "now"}` -> `(2026-04-07T14:00, None)`
- `{after: "2026-04-01", before: "2026-04-14"}` -> `(2026-04-01T00:00, 2026-04-15T00:00)` (both inclusive per RESOLVE-10)

**RESOLVE-05 ("now" consistency):** The resolver takes `now` as a parameter. The caller (Phase 46 pipeline) passes the same `now` to all resolve_date_filter calls in a single query. The resolver itself doesn't need to enforce this -- it's the caller's contract.

**Key design decisions for "this" month/year:**
- `{this: "m"}` uses calendar-aware boundaries: first of current month to first of next month (NOT first + 30d). Per RESEARCH.md Open Question 2 recommendation.
- `{this: "y"}` uses calendar-aware boundaries: Jan 1 of current year to Jan 1 of next year.
- Naive 30d/365d approximation applies to `{last: "Nm"}` and `{next: "Nm"}` where the anchor is "now" and the range is approximate.

def _resolve_this(unit: str, now: datetime, *, week_start: int) -> tuple[datetime, datetime]:
    today = _midnight(now)
    if unit == "d": return (today, today + timedelta(days=1))
    if unit == "w":
        days_since_start = (today.weekday() - week_start) % 7
        start = today - timedelta(days=days_since_start)
        return (start, start + timedelta(days=7))
    if unit == "m":
        start = today.replace(day=1)
        # Calendar-aware: first of next month
        if start.month == 12:
            end = start.replace(year=start.year + 1, month=1)
        else:
            end = start.replace(month=start.month + 1)
        return (start, end)
    if unit == "y":
        start = today.replace(month=1, day=1)
        end = start.replace(year=start.year + 1)
        return (start, end)

def _resolve_last(duration: str, now: datetime) -> tuple[datetime, datetime]:
    count, unit = _parse_duration(duration)
    delta = _duration_to_timedelta(count, unit)
    start = _midnight(now - delta)
    return (start, now)

    count, unit = _parse_duration(duration)
    delta = _duration_to_timedelta(count, unit)
