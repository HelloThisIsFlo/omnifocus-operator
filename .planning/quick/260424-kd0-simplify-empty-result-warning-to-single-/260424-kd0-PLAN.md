<objective>
Collapse the parameterized `EMPTY_RESULT_WARNING_{SINGLE,MULTI}` surface (shipped in quick task 260424-j63) to a single static constant. Remove all filter-name enumeration, `is_non_default` iteration at the warning call site, camelCase alias lookups, alphabetical sorting, and the zero-filter skip. The warning now fires unconditionally when `list_tasks` returns zero items.

Purpose: The live-probe surfaced behaviors where pagination and envelope fields (`limit`, `offset`, `include`, `only`) were being listed as "filters", creating misleading warnings. The parameterized surface also carried complexity (~70 LOC) the agent didn't use semantically. The static nudge is equivalent to the agent, simpler to maintain, and eliminates the whole machinery around active-filter tracking.

Output: A single static `EMPTY_RESULT_WARNING` constant, a rewritten emit helper, a collapsed test matrix (9→3), four updated collateral assertions, cleaned docstring trails, annotated planning artifacts, and the source todo moved to `completed/`.
