# v1.2 Combined Regression Suite

Comprehensive regression covering all Milestone v1.2 features: read lookups, task creation, field editing, tag operations, move operations, and lifecycle management. 87 tests total.

## Conventions

- **Inbox only.** Never create tasks in projects. Every task goes to inbox (no `parent` for top-level, or under the test parent task).
- **Timezone required.** Date fields need timezone info in ISO 8601 (e.g., `+01:00` or `Z`). Without it, Pydantic rejects the value.
- **Tags by ID.** Some tag names are ambiguous. Discover tags via `get_all` first, then use IDs where names might collide.
- **1-item limit.** Both `add_tasks` and `edit_tasks` currently accept exactly 1 item per call.
- **No parallel error calls.** Claude Code cancels all sibling calls when one errors. Run error-expecting calls individually.

**Known issue — spurious "no changes" warning:** When an action results in a no-op AND no field edits are in the request, a spurious "No changes specified/detected" warning fires alongside the specific warning. Document in the report's Observations section if encountered.

### Step 1 — Discover Tags & Entities

Call `get_all` and extract:
- **Tags:** Pick 3 tags with unique names (no ambiguity). Store as tag-a, tag-b, tag-c (IDs and names). Check if any tag name maps to 2+ tag IDs (for ambiguity test). Store the ambiguous name if found.
- **Projects:** Pick 1 real project. Store its ID and name as proj-a.
- **Tags (for lookups):** Store 1 tag ID as tag-lookup.

### Step 3 — Manual Actions
