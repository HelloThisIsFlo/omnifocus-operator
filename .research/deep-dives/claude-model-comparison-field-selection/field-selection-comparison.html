Dynamic Schema Relaxation

The MCP Paradox

If a tool defines a field as **required**, MCP clients will fail to validate the response if it's projected out.

The winner solves this by **patching the schema at registration**, keeping fields documented but making them optional for the protocol.

Standard Pydantic Schema = Rigid
Patched Winner Schema = Flexible
projection.py 
# The Secret Sauce
def build_projected_schema(item_type: type):
    schema = ListResult[item_type].model_json_schema()
    schema = copy.deepcopy(schema)

    item_name = item_type.__name__
    defs = schema.get("$defs", {})
    if item_name in defs:
        defs[item_name]["required"] = ["id"]

    return schema
