Three-Layer Flow
Every request flows top-to-bottom through three layers. Each layer is a Python protocol (structural typing), so implementations are swappable without inheritance.
Layer 1: MCP Server
server.py — Tool registration, lifespan management, stdio transport. Thin: delegates everything to service.
Layer 2: Service
service.py — Validation, tag/parent resolution, cycle detection, no-op detection, warnings. Business logic lives here.
Layer 3: Repository
repository/ — Data access protocol with 3 implementations. Reads from SQLite or bridge, writes always via bridge.
Service Layer
OperatorService
Wraps the Repository protocol. Read methods delegate directly. Write methods add validation and resolution.
add_task()
Validates name → resolves parent (project or task) → resolves tag names to IDs (case-insensitive) → delegates to repository
edit_task()
Existence check → builds minimal patch (only non-UNSET fields) → handles tag modes (replace/add/remove) → no-op detection → cycle detection → warnings for completed/dropped
_resolve_tags()
Case-insensitive name lookup with fallback to ID lookup. Returns resolved tag IDs.
_check_cycle()
Walks parent chain to detect circular references when moving tasks.
ErrorOperatorService
Degraded mode: server stays alive, serves startup error on any attribute access. Bypasses normal init via object.__setattr__().
Repository Layer
Repository protocol
Structural typing via @runtime_checkable Protocol. Methods: get_all(), get_task(), get_project(), get_tag(), add_task(), edit_task()
HybridRepository
Default. Reads from OmniFocus SQLite cache (~46ms). Writes via bridge, then marks cache stale + waits for fresh WAL mtime (2s timeout, 50ms poll). Complex SQL with joins, timezone handling, status mapping via lookup tables.
production
BridgeRepository
Bridge-based with async lock serializing all reads. Cache invalidation via MtimeSource. Concurrent reads share a single refresh. Fallback option via OMNIFOCUS_REPOSITORY=bridge-only.
fallback
InMemoryRepository
Pre-built snapshot for testing. Simple linear lookups. Write operations mutate the snapshot directly.
testing only
Bridge Layer
Bridge protocol
Single method: async send_command(operation: str, params: dict) → dict. Three implementations communicate with OmniFocus in different ways.
RealBridge
File-based IPC: writes JSON request to ~/Library/.../ipc/, triggers OmniFocus via omnifocus:///omnijs-run URL scheme, polls for response (50ms sleep, 10s timeout). Atomic writes via .tmp + os.replace().
SimulatorBridge
Extends RealBridge, overrides _trigger_omnifocus() as no-op. Allows mock simulator process to respond via same IPC files.
InMemoryBridge
Test double with injected data, call tracking via BridgeCall records, and configurable error simulation.
Adapter layer: adapter.py transforms bridge responses into model shape. Dict-based mapping tables with fail-fast on unknown values. Maps task status → (urgency, availability) tuple.

Write Models
Uses UNSET sentinel (singleton _Unset) for patch semantics — distinguishes "field omitted" from "field set to null".
TaskCreateSpec
name (required) + optional parent, tags, dates, flagged, estimatedMinutes, note
TaskEditSpec
id (required), all others UNSET-defaulted. Tag modes: replace vs add/remove (mutually exclusive). MoveToSpec for repositioning.
Safety & Config
SAFE-01: No automated test, CI, or agent touches RealBridge. The bridge factory raises RuntimeError when PYTEST_CURRENT_TEST is set. All automated testing uses InMemoryBridge or SimulatorBridge.

SAFE-02: RealBridge interaction is manual UAT only, run by the human user against their live OmniFocus database. UAT scripts live in uat/ and are excluded from pytest and CI.
Environment Variables
Repository and bridge routing via env vars. Factory pattern with validation.
OMNIFOCUS_REPOSITORY
Default: hybrid. Options: hybrid, bridge-only
OMNIFOCUS_BRIDGE
Default: real. Options: real, simulator, inmemory
OMNIFOCUS_SQLITE_PATH
SQLite cache path for HybridRepository reads
OMNIFOCUS_BRIDGE_TIMEOUT
Bridge operation timeout in seconds (default: 10)
Key Technical Patterns
  • Structural typing — Bridge, Repository, MtimeSource are all @runtime_checkable Protocol
  • Factory routing — create_bridge() and create_repository() with env var control
  • Async safety — asyncio.to_thread() wraps blocking I/O; asyncio.Lock() serializes concurrent refreshes
  • Cache invalidation — WAL-based mtime tracking (50ms poll, 2s timeout after writes)
  • Graceful degradation — ErrorOperatorService keeps server alive on startup failure
  • Single runtime dep — mcp>=1.26.0 (uses built-in FastMCP, not standalone package)
