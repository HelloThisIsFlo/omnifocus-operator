Service Layer

omnifocus_operator / service.py

The validation and business logic layer sitting between MCP tools and the Repository protocol. Read methods are pure passthroughs. Write methods add name validation, tag/parent resolution, cycle detection, no-op detection, and warnings.

add_task Flow
add_task(spec: TaskCreateSpec) → TaskCreateResult
Three validation steps before repository delegation. Fails fast on any error.
1
Name Validation
Rejects empty strings and whitespace-only names.
ValueError("Task name is required")
2
Parent Resolution if provided
Calls _resolve_parent(spec.parent). Checks if parent exists as a project first, then as a task.
ValueError("Parent not found: ...") if neither.
3
Tag Resolution if provided
Calls _resolve_tags(spec.tags). Case-insensitive name match, falls back to ID lookup. Resolved IDs passed as resolved_tag_ids keyword arg.
4
Repository Delegation
repository.add_task(spec, resolved_tag_ids=ids) → TaskCreateResult(success, id, name)
edit_task Flow
edit_task(spec: TaskEditSpec) → TaskEditResult
The most complex method. Nine-step pipeline with validation, payload construction, tag modes, movement, cycle detection, and no-op detection.
1
Task Existence
Fetches task by spec.id. If not found: ValueError("Task not found: ...")
2
Availability Warning
If task is completed or dropped, appends warning (changes still applied).
3
Name Validation if set
Only validates when name is not UNSET. Empty/whitespace: ValueError("Task name cannot be empty")
4
Build Payload — Simple Fields
Maps non-UNSET fields to camelCase payload: name, flagged, note (None → ""), estimatedMinutes
5
Build Payload — Dates
Non-UNSET date fields serialized to ISO 8601 or None (clears). Keys: dueDate, deferDate, plannedDate
6
Tag Mode Resolution
Detects which mutually-exclusive tag mode is active: replace, add, remove, or add+remove. Resolves tag names → IDs, generates duplicate/absent warnings. See Tag Modes section.
7
Movement & Cycle Detection
Processes moveTo. Resolves containers via _resolve_parent(), anchors via get_task(). If moving to a task container, runs _check_cycle(). See Movement section.
8
No-Op Detection
8a. Empty edit: payload has only id key → returns early with "No changes specified" warning.
8b. All fields unchanged: compares payload against current task state, sorted tag IDs → "No changes detected".
9
Repository Delegation
repository.edit_task(payload) → TaskEditResult(success, id, name, warnings?)

Design choice: Tag resolution accepts both names and IDs transparently. The agent can pass human-readable tag names ("Work", "errands") and the service resolves them case-insensitively. If no name matches, it tries the input as a tag ID — so raw IDs also work without the agent needing to know which format it's using.
Tag Modes
Four Mutually Exclusive Tag Modes
The tags field (replace) is mutually exclusive with addTags/removeTags (incremental). The service detects the mode from which fields are non-UNSET.
Mode Fields Set Payload Behavior
replace tags tagMode: "replace"
tagIds: [...]		 Full replacement. tags: null clears all tags.
add addTags only tagMode: "add"
tagIds: [...]		 Appends. Warns if tag already on task.
remove removeTags only tagMode: "remove"
removeTagIds: [...]		 Removes. Warns if tag not on task.
add + remove addTags + removeTags tagMode: "add_remove"
addTagIds + removeTagIds		 Both in one call. Warns on duplicate add or absent remove.
Movement & Cycles
MoveToSpec — exactly one key
Four position types with different validation paths. Pydantic validator _exactly_one_key() ensures mutual exclusivity.
Container Positions
beginning / ending
Value = project ID, task ID, or None (inbox)

• Resolves via _resolve_parent()
• If container is a task: runs _check_cycle()
• If None: moves to inbox
Anchor Positions
before / after
Value = sibling task ID

• Validates anchor task exists
• ValueError("Anchor task not found") if missing
• No cycle check needed (sibling, not parent)

Write Models
The UNSET Sentinel
Singleton _Unset class distinguishes three field states for patch semantics. JSON schema cleaning removes UNSET references for clean MCP tool schemas.
UNSET
Field omitted by caller
no change
None
Explicitly set to null
clear value
"Buy milk"
Set to a value
update field
TaskCreateSpec
  • name: str — required
  • parent: str | None — project or task ID
  • tags: list[str] | None — names or IDs
  • due_date: AwareDatetime | None
  • defer_date: AwareDatetime | None
  • planned_date: AwareDatetime | None
  • flagged: bool | None
  • estimated_minutes: float | None
  • note: str | None
TaskEditSpec
  • id: str — required
  • name, flagged — value-only (cannot clear)
  • note, due_date, defer_date, planned_date, estimated_minutes — clearable (None = clear)
  • tags — replace mode
  • add_tags, remove_tags — incremental mode
  • move_to: MoveToSpec — exactly one of beginning/ending/before/after
All optional fields default to UNSET.
Degraded Mode
ErrorOperatorService
Created when server startup fails (e.g. OmniFocus not found, SQLite path invalid). Keeps the MCP server alive so the agent gets a useful error message instead of a connection failure.
Construction
Receives the startup Exception. Uses object.__setattr__() to bypass normal __init__ and store the error message directly.
__getattr__
Intercepts every attribute access. Logs a warning and raises RuntimeError with the startup error and restart instructions.
Effect
Any MCP tool call → service.get_all_data() → triggers __getattr__ → agent sees error explaining what went wrong.
Integration: The MCP server's app_lifespan context manager creates OperatorService(repository) on success and ErrorOperatorService(exc) on failure. Both are stored as ctx.lifespan_context["service"] and accessed identically by tool handlers — duck typing means the tools don't know which one they're talking to until they try.
